fix(spot-brief): 한글 키워드 이중 URL 인코딩 차단

httpClient.request(String, ...) 는 RestTemplate#exchange 의 URI 템플릿으로
해석되어 %-인코딩된 한글 키워드의 % 가 %25 로 재인코딩됐다. 이로 인해 KTO
searchKeyword2 가 키워드를 인식 못 하고 totalCount=0 을 반환, 모달의 brief
가 항상 null 로 비어 있었다.

URI.create(url) 로 감싸 requestUri(URI, ...) 를 호출하도록 변경해 재인코딩을
차단한다 (다른 KTO 어댑터들이 이미 사용 중인 패턴).
