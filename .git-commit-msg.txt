perf(camping): 상세→뒤로가기 시 검은 화면/3-4초 지연 제거 (sessionStorage hydration)

문제
- /camping 카드 → 상세 → 뒤로가기 시 3~4초간 검은 화면이 보였다가 목록이 다시 그려짐
- /camping 은 "use client" 페이지라 라우트 이탈 시 CampingInner 가 unmount → 뒤로가기 시
  state(sites/visibleCount/keyword/scrollY) 가 모두 초기화되고 useEffect 가
  /api/v1/camping?limit=0 (전국 ~3,700건, ~3MB JSON) 을 처음부터 재호출
- 그 사이 Suspense fallback 이 plain "Loading…" 텍스트뿐이라 body 배경(#09090b)
  이 그대로 노출되어 "검은 화면" 인상이 남음

수정 (netplix-frontend/app/camping/page.js)
- sessionStorage 5분 TTL 캐시 (LIST_CACHE_KEY = camping_list_cache_v1)
  · 검색 컨텍스트(keyword + nearbyMode) 별로 sites / visibleCount / scrollY 저장
  · readListCache / writeListCache / patchListCache 헬퍼
- mount 시 캐시 hit → 즉시 hydrate (skeleton/fetch 모두 0회)
  · useState 초기값으로 sites/loading=false/visibleCount/nearbyMode 박음
  · skipFetchOnceRef 로 첫 useEffect 의 fetch 를 건너뜀 (어떤 이유로든
    첫 effect 실행 시 즉시 ref 소비 → 키워드 변경 등 후속 동작은 정상 fetch)
- 카드 클릭 직전 window.scrollY 를 캐시에 박아두고 router.push
  · goDetail() 헬퍼로 onClick / onKeyDown(Enter, Space) 공통 처리
- 돌아오면 mount 직후 2× requestAnimationFrame 뒤 scrollTo 로 위치 복원
  · CSS scroll-behavior: smooth 의 가로채기를 막기 위해 잠깐 auto 로 강제 후 원복
- ?nearby=true 자동 트리거 가드: 캐시가 nearby 모드로 hydrate 된 상태면
  handleNearby()(GPS 재요청 + API) 호출도 생략 (skipAutoNearbyRef)
- visibleCount 변동 시 캐시 동기화 → "더 보기" 누른 상태도 그대로 복원
- Suspense fallback 을 CampingRouteFallback 으로 교체
  · 본문(.cmp-root) 과 동일한 그라데이션 + hero 자리표시자 + 8개 카드 스켈레톤
  · 라우트 전환 도중에도 톤 일치 → 검은 화면 인상 제거

동작
- /camping → 카드 → 뒤로가기 (5분 이내): 즉시 직전 화면 그대로 복원, fetch 0회
- /camping?q=서울 → 카드 → 뒤로가기: 키워드별 캐시 분리, 즉시 복원
- /camping?nearby=true → 카드 → 뒤로가기: GPS/API 호출 0회로 즉시 복원
- TTL 만료 / 키워드 변경 시: 정상 fetch 동작 유지
